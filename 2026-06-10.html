01

MCTS — 首个 MCP 安全扫描器

MCP 服务器数量突破 10,000 后，安全从"最好有"变成了"必须有"。MCTS（Model Context Threat Scanner）是第一个专为 MCP 设计的本地安全扫描器。

核心信息

仓库MCP-Audit/MCTS
Stars⭐21
语言Python
判标⚡ 值得关注进度

为什么重要：Agent 通过 MCP 获得的能力越来越强——文件读写、shell 执行、网络请求、数据库访问。每一个 MCP server 都是一个潜在攻击面。但在此之前，MCP 安全几乎完全空白——没有扫描器，没有审计工具，没有安全评分。MCTS 填补了这个关键空白。

三大特性：

  • 本地优先（不依赖云服务，数据不出本机）
  • 多种分析器：schema 校验、权限审计、代码扫描
  • 输出可审计的安全报告

同类信号：AgentSploit 在 HN 同期出现，定位为"Burp Suite for AI Agents and MCP Servers"，进一步确认 Agent 安全赛道的形成。两个工具同一周上线不是巧合——是需求到了。

判标说明：⚡ 值得关注进度。方向极其重要，填补了 MCP 生态的空白拼图。但极早期（21 stars，创建仅 7 天），功能还需迭代。给一个月——届时再看社区反馈和功能完整度。
02

Agent Manager — 桌面端 Agent & MCP 管理器

告别 CLI 敲命令管理 agent 和 MCP 服务器。Agent Manager 用 Electron 提供了图形化的统一管理体验。

核心信息

仓库Zafer-Liu/Agent_Manager
Stars⭐136
语言TypeScript (Electron)
判标⚡ 值得关注进度

为什么重要：Agent 工具栈一直以 CLI 为主。要装 MCP server 得手动编辑 yaml，要看 agent 状态得读日志。Agent Manager 把 agent 运维带入 GUI 时代：可视化配置、一键启动/停止、状态监控。降低了非开发者的使用门槛，也让专业用户能一目了然看到所有 agent 的运行状态。

同类信号正在形成品类：

  • getgantry/gantry — macOS Docker GUI + 内置 MCP server
  • farion1231/cc-switch ⭐97K — 全平台 agent 切换器，支持 Claude Code / Codex / OpenCode / Hermes 等
判标说明：⚡ 值得关注进度。方向清晰——agent 管理需要 GUI，136 stars / 5 天前的势头很好。但功能极早期，兼容性和安装体验还需迭代。这和 1990 年代 Linux 从终端到 GNOME 的路径如出一辙——基础设施下沉的标志。
