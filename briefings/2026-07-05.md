### ① MCP 2026-07-28 协议大变革 — Agent 基础设施的"无状态"拐点
MCP 团队于 5 月 21 日锁定 2026-07-28 Release Candidate，这是 MCP 自发布以来最大的一次协议修订。6 大 Breaking Changes 同时抵达：协议层彻底无状态化（不再需要 session）、Extensions 框架（MCP Apps + Tasks）、授权加固、正式弃用策略。7 月 28 日最终规范发布，所有生产 MCP 服务器需要在此之前完成迁移。

🔥 **值得马上试** — 如果你维护 MCP 服务器，现在就需要读 RC 并制定迁移计划。无状态化将根本性简化部署架构——不再需要 sticky session 路由和共享 session 存储。

## 📊 趋势洞察

### 趋势 1：MCP 进入"基础设施化"阶段——无状态协议改写部署模式

MCP 2026-07-28 RC 是 MCP 从"协议"到"基础设施"的关键转折点。无状态化意味着 MCP 服务器可以像普通 HTTP 服务一样部署——不需要 sticky session、不需要共享 session store、不需要复杂的网关配置。一个之前需要 Redis session store + L7 负载均衡的 MCP 部署，现在可以在 round-robin 负载均衡后面直接跑。

这与 78% 企业 AI 团队已在生产中使用 MCP 的数据相呼应：MCP 正在从"实验性协议"变成"无聊但可靠的基础设施层"——就像 HTTP 和 TCP 一样。接下来的 3-4 周（7 月 28 日前）将是 MCP 服务器大规模迁移的窗口期。

**传导链：** 无状态协议 → 部署简化 → 企业采纳加速 → MCP 网关/安全/治理品类爆发 → MCP 从可选变成默认。

### 趋势 2：Agent 基础设施品类爆发——网关、测试、协作三层同时在建

本周 GitHub Trending 集中出现了多个 Agent 基础设施品类：
- **网关层**：OmniRoute（231+ 提供商单端点路由）、Tokenwise（LLM 代理，发现超付）
- **测试/沙盒层**：TryCase（agent 一次性测试环境）、Termi Protocol（agent 3D 可视化执行）
- **协作层**：CircleChat（agent 多对多协作）、N71（跨 agent 共享上下文）
- **浏览器层**：BrowserAct（agent 专用浏览器自动化）、Safari MCP Server（平台级浏览器集成）

这些不是"agent 版 X"之类的替代品，而是专门为 agent 设计的原生基础设施。当 agent 从"偶尔用一下"变成"基础设施上运行的持续性工作负载"时，agent 网关、agent 测试、agent 协作都是独立的品类机会。

**传导链：** Agent 使用频率↑ → 需要专门的网关/测试/协作层 → Agent 基础设施品类从零到一。下半年的竞争不是"谁做了更好的 agent"，而是"谁建了更好的 agent 基础设施"。

## 💎 本周最值得试的新技能

**给 MCP 服务器维护者：读 2026-07-28 RC 并开始迁移规划**

理由是 7 月 28 日最终 spec 发布，如果你等到那时才开始迁移，会很赶。现在就用 RC 做兼容测试是最稳妥的策略。重点关注的 Breaking Changes：
1. **Session 移除**：每个请求自包含 `_meta` 字段，不再需要 session handshake
2. **新 HTTP 请求头**：`Mcp-Method` 和 `Mcp-Name` 为必需头
3. **Auth 授权加固**：更强的权限边界

参考：[ChatForest 迁移指南](https://chatforest.com/builders-log/mcp-spec-2026-07-28-release-candidate-stateless-breaking-changes-builder-guide)

**给 web 开发者：接入 Safari MCP Server**

WebKit 官方发布的 MCP 服务器，质量有保障。让 AI agent 驱动浏览器不再需要第三方工具——性能和兼容性都是原生级别。

**关注 vercel-labs/skills 和 cloudflare/skills**

Platform Skills 矩阵从 4 家（MS/Google/Anthropic/Vercel）扩展到 6 家。Vercel 的 skills 覆盖前端部署和 edge functions，Cloudflare 的 skills 覆盖 Workers 和边缘计算场景——这两个领域之前缺乏官方 skills 覆盖，现在被补齐了。
