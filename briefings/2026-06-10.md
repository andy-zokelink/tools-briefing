## ⚡ 本期重点推荐

### ① MCTS — 首个 MCP 安全扫描器，本地静态分析 + 实时工具发现
> MCP 服务器数量突破 10,000 后，安全从"最好有"变成了"必须有"。MCTS 是第一个专为 MCP 设计的本地安全扫描器。

- 仓库：[MCP-Audit/MCTS](https://github.com/MCP-Audit/MCTS) ⭐21
- 定位：MCP 生态的首个专用安全扫描工具。本地优先（不依赖云服务），支持静态分析 MCP 配置文件 + 实时工具发现时的安全评估，多种分析器（schema 校验、权限审计、代码扫描），输出可审计报告。
- 为什么重要：MCP 服务器注册量破万，但安全层几乎空白。Agent 通过 MCP 获得文件读写、shell 执行、网络请求等能力——每一个 MCP server 都是一个潜在攻击面。MCTS 填补了这个关键空白。它的出现标志 MCP 生态从「野蛮生长」进入「安全治理」阶段。
- 同类信号：AgentSploit（HN 同期出现，定位"Burp Suite for AI Agents and MCP Servers"）进一步确认了 Agent 安全赛道的形成。
- 状态判断：⚡ 值得关注进度 — 方向极其重要，填补了 MCP 生态的空白拼图。但处于极早期（21 stars，创建仅7天），功能成熟度和社区验证还需观察。
- 数据源：GitHub API, Hacker News

### ② Agent Manager — 首款跨平台桌面端 Agent & MCP 管理器
> 告别 CLI 敲命令管理 agent 和 MCP 服务器。Agent Manager 用 Electron 提供了图形化的 agent/MCP 管理体验。

- 仓库：[Zafer-Liu/Agent_Manager](https://github.com/Zafer-Liu/Agent_Manager) ⭐136
- 定位：跨平台桌面应用（Electron + TypeScript），统一管理你的 AI Agent 和 MCP 服务器。可视化配置、启动/停止、状态监控。
- 为什么重要：Agent 工具栈一直以 CLI 为主——要装 MCP server 得手动编辑 yaml，要看 agent 状态得看日志。Agent Manager 把 agent 运维带入 GUI 时代，降低了非开发者的使用门槛，也让专业用户能一目了然看到所有 agent 的运行状态。这是 agent 工具从"开发者专属"走向"全用户可用"的标志性一步。
- 同类信号：`getgantry/gantry`（macOS Docker GUI + 内置 MCP server）、`cc-switch`（全平台 agent 切换器 ⭐97K）——桌面端 agent 工具正在形成一个新品类。
- 状态判断：⚡ 值得关注进度 — 方向清晰（agent 管理需要 GUI），136 stars / 5 天前创建的势头不错。但功能还在极早期，安装体验和兼容性需要迭代。一个月后再看。
- 数据源：GitHub API

## 📊 趋势洞察

### 1. MCP 安全从"真空"到"元年"——两个工具同周上线不是巧合
MCTS 和 AgentSploit 在同一周出现在 GitHub 和 HN 上，不是偶然。MCP 注册量破万 → agent 通过 MCP 获得的能力越来越强（文件系统、shell、网络） → 攻击面急剧扩大 → 安全工具需求爆发。

> 传导链：MCP 标准化 → 工具数破万 → Agent 获得高危权限 → 安全真空暴露 → 安全扫描/渗透测试工具涌现

2026 下半年预判：MCP 安全将成为独立赛道。预计出现"可信 MCP server 认证"、"MCP server 安全评分"、"企业 MCP 防火墙"等产品形态。

### 2. Agent 工具从 CLI 走向桌面——"Agent OS"的雏形
Agent_Manager、gantry、cc-switch 的同期出现，指向同一个趋势：agent 管理正在从"开发者敲命令"走向"打开桌面应用"。这和 1990 年代 Linux 从终端到 GNOME/KDE 的路径如出一辙。

> 信号：agent 工具的使用者正在从"写代码的人"扩展到"用 agent 干活的人"——后者不需要会 yaml 语法和 CLI 参数，只需要一个看得懂的界面。

## 💎 本周最值得试的新技能

用 **MCTS** 扫描你当前的 MCP 配置，看看有多少安全隐患：

```bash
# 安装
git clone https://github.com/MCP-Audit/MCTS.git
cd MCTS
pip install -r requirements.txt

# 扫描你的 MCP 配置
python mcts.py --config ~/.hermes/profiles/default/config.yaml
```

替代方案：如果你管理多个 MCP server，装 **Agent Manager** 试试图形化管理体验——虽然还在早期，但方向对。
