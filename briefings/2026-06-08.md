## ⚡ 本期重点推荐

### ① headroom — LLM 上下文压缩，省 60-95% token
> 把工具输出/日志/RAG 片段在送进 LLM 前先压缩 60-95%，支持 library / proxy / MCP server 三种模式。

- 仓库：[chopratejas/headroom](https://github.com/chopratejas/headroom) ⭐14,148
- 定位：LLM 上下文压缩器。在工具输出进入模型前自动压缩，砍掉冗余日志、空行、重复结构，保留语义信息。
- 为什么重要：Agent 调用工具越多 → 输出数据越多 → 上下文窗口越贵。headroom 直接砍掉 60-95% 的冗余，对 Hermes、Claude Code、Codex 这类频繁调用工具的 coding agent 效果立竿见影。
- 三种模式：library（Python pip install）、proxy（HTTP 中间层拦截）、MCP server（挂到 Agent 的 MCP 配置）
- 状态判断：🔥 值得马上试 — #1 Trending 当周，社区验证充分，接入成本极低（MCP server 模式一个 yaml 配置）
- 数据源：CSDN GitHub 日榜 2026-06-04, GitHub API

### ② harness — 元技能：自动设计 Agent 团队并生成技能
> 给一个任务描述，harness 自动分析需要哪些专业 agent、每个 agent 用什么技能、skill 文件该写什么——然后直接生成。

- 仓库：[revfactory/harness](https://github.com/revfactory/harness) ⭐6,096
- 定位：Meta-skill——生成 skills 的 skill。从任务描述出发，自动拆解为 agent 团队结构+各 agent 的技能清单+skill 文件内容。
- 为什么重要：对 Skills Watch 来说是革命性的。skills 不再靠人手写，而是由 meta-skill 自动生成。Agent 技能生态即将从"手工时代"进入"工业化时代"。
- 与 Hermes 的关系：可以试着让它分析你的日常工作，自动生成一套 Hermes skill
- 状态判断：🔥 值得马上试 — 概念验证通过，方向正确，但从生成到好用还需要迭代
- 数据源：CSDN GitHub 日榜, GitHub API

## 📊 趋势洞察

### 1. 上下文窗口成为 Agent 新瓶颈，压缩方案集中爆发
headroom（#1 Trending）、codegraph（预索引减少 token）、claude-mem（记忆引擎）、stop-slop（消除冗余输出）本周齐头并进。工具调用生态越丰富，输出数据越多，上下文越贵。

> 传导链：MCP 标准化 → 工具数爆发 → 单次调用输出量暴增 → 上下文窗口压力 → 压缩/摘要/记忆成为刚需

行业共识：2026 年 Agent 的核心瓶颈已从"能调用多少工具"转移到"调用后能塞进多少上下文"。这不再是锦上添花——是刚需。

### 2. GitHub 正被 AI Agent 压垮
2026 年单周提交量 2.75 亿次，是 2025 全年 10 亿次的 14 倍年化率。GitHub Actions 从每周 5 亿分钟（2023）飙升至 21 亿分钟（2026 初）。Claude Code 单一工具贡献全平台公开仓库提交量的 4.5%——每周 260 万次，较半年前增长 25 倍。

> 信号：AI Agent 正在成为 GitHub 上最活跃的"用户"。这既证明了 agentic coding 的爆发，也暴露了软件基础设施应对 AI 规模化的深层压力。GitHub 在 2026 年初频繁宕机不是意外，是结构性挑战。

数据源：搜狐/网易 2026-06-07 报道，GitHub 官方数据

### 3. Agent Skills 进入「元技能」+「大厂入局」双轮驱动
harness 的出现标志着 Skills 从手工创作进入自动生成时代。同时 Microsoft Azure Agent Skills 发布、agentskills.io 获 7+ 平台适配，说明大厂正在将 Skills 视为 Agent 生态的入口标准。结合 MCP 的 Linux Foundation 托管，整个 Agent 互联的基础协议层正在快速固化。

> 下半年预期：出现"技能市场/技能商店"，像 App Store 之于手机一样，Skills Store 之于 Agent。

## 💎 本周最值得试的新技能

在 Hermes 上装 `headroom` 的 MCP server 模式——把它挂到你的 MCP 配置里，所有工具输出自动压缩后再进 LLM。token 钱省多少不说，关键是上下文窗口腾出来给真正重要的内容。

```bash
# 安装
pip install headroom

# 或 MCP server 模式配置（添加到 Hermes MCP config）
# servers:
#   headroom:
#     command: headroom
#     args: ["serve"]
```

替代方案：如果你更关心代码理解，试 `codegraph`（`npx codegraph`）对本地仓库预索引，减少 agent 重复读文件的 token 消耗。
