## ⚡ 本期重点推荐

### ① sandboxd — AI Agent 的即时沙箱环境
> 一个命令启动隔离开发环境，带预览 URL，专为 coding agent 设计。无需 Kubernetes。

- 仓库：[tastyeffectco/sandboxd](https://github.com/tastyeffectco/sandboxd) ⭐503
- 定位：为 AI coding agent 提供开箱即用的隔离执行环境。`npx sandboxd` 一条命令即可获得沙箱+预览 URL。
- 为什么重要：对 Hermes Agent / Claude Code 这类 agent 来说，最头疼的就是「安全隔离执行环境」。sandboxd 把这个成本降到一行命令，比 Docker 轻量得多，不需要 K8s。
- 状态判断：⚡ 值得关注进度 — 刚发布，生态还不够成熟，但方向对。等社区反馈和稳定性验证后再深入集成。
- 数据源：GitHub Trending (weekly)

### ② Lexa — 把代码库变成 AI 可查询的图
> Rust 写的本地代码智能引擎，将整个代码库转换为可移植、可查询的图结构。

- 仓库：[anvia-hq/lexa](https://github.com/anvia-hq/lexa) ⭐83
- 定位：给 AI agent 当「代码库眼睛」。把代码库静态分析为图结构（AST、调用关系、依赖图），agent 可以查询「这个函数被哪些模块调用」「这个类型的完整定义链」。
- 为什么重要：目前的 agent 读代码靠 grep + 文件读取，Lexa 提供结构化理解。跟 Hermes 的代码理解能力天然互补。
- 状态判断：🔥 值得马上试 — 直接在 Hermes 项目上跑一下看效果。83 stars 说明刚出来，但概念验证很强。
- 数据源：GitHub Trending (daily)

## 📊 趋势洞察

### 1.「Agent 基础设施」从推理转向执行环境
今年上半年 MCP 协议让 agent 能「看到」更多工具，但现在瓶颈转移到「在哪跑」。sandboxd、Daytona、Modal 这些隔离执行方案开始密集出现，说明行业共识：工具调用问题基本解决，执行安全问题成为新瓶颈。
> 传导链：MCP 标准化工具接口 → 工具数量爆发 → agent 需要安全执行环境 → sandboxd 等沙箱方案崛起

### 2. MCP server 从「能用」转向「好用」
从最初的 JSON-RPC 包装器，到现在 Rust 原生亚毫秒级实现（opendocswork-mcp）、专有硬件接入（totem/Whoop）、GUI 工具链（Burrow），MCP 生态质量在显著提升。
> 信号：Rust MCP server 出现 → 性能成为差异化因素；硬件接入 MCP → 生态从纯软件扩展到 IoT/健康

## 💎 本周最值得试的新技能

装个 `sandboxd` 给 Hermes 当沙箱——一行命令启动隔离环境，agent 在里面随便折腾不影响主机。对本地跑 coding agent 来说，这比 Docker 轻量得多。

```bash
npx sandboxd
```
