## ⚡ 本期重点推荐

### ① Ratel — 给 Agent 无限工具和技能，不爆上下文窗口

Ratel 解决了一个 Agent Skills 生态的核心痛点：每装一个 skill 就吃掉一部分上下文窗口，skill 越多越笨。Ratel 通过按需动态加载（just-in-time tool retrieval）让 agent 拥有"无限技能"而上下文成本恒定。

核心机制：Ratel 不把全部 tool/skill schema 塞进 system prompt，而是运行时根据用户意图检索和注入相关工具。这意味着你可以装 1000 个 skills/MCP servers 而不影响推理质量。

🔥 **值得马上试** — 如果你在用 Claude Code / Hermes Agent 且 skills 超过 10 个，上下文膨胀已经是肉眼可见的问题。Ratel 给了一条现实的缓解路径。
- HN 讨论：23 points, 18 comments
- [github.com/ratel-ai/ratel](https://github.com/ratel-ai/ratel)

### ② Microsoft Flint — 微软发布 AI Agent 可视化语言

微软正式发布 Flint，一种专为 AI Agent 设计的可视化编程语言。不同于传统图表工具，Flint 的设计目标是被 AI agent 生成和消费——agent 用 Flint 表达推理过程、数据流和决策树，人类用 Flint 审查和理解 agent 行为。

Flint 的核心创新在于它是"agent-native"的：语法设计为 LLM 友好（结构化、可验证、zero-shot 生成准确率高），同时保持人类可读。这标志着 Agent 可解释性从"事后日志"走向"结构化第一性语言"。

⚡ **值得关注进度** — 方向极对（agent 可解释性是刚需），但刚发布仍需观察生态适配（IDE 集成、MCP server 支持、与主流 agent 框架的互操作）。
- HN #1：278 points, 119 comments
- [microsoft.github.io/flint-chart](https://microsoft.github.io/flint-chart/#/)

## 📊 趋势洞察

### 趋势 1：Agent 上下文管理成为本周最热技术主题

本周三个独立项目从不同角度解决同一问题——agent 上下文窗口不够用：

- **Ratel**：按需 tool/skill 注入，不预装全部 schema
- **deja-vu**：通过 SSH 做 agent 编码记忆跨会话持久化
- **ContextVault**：团队共享的 AI 记忆层

加上上周的 **TencentDB-Agent-Memory**（4 级本地持久化记忆），"agent 记忆管理"正在从"nice to have"变成基础设施竞赛。信号非常明确：agent 不是缺 skills，是缺少让 skills 不互相打架的记忆架构。

传导链：单 agent 无记忆 → 单 agent + 本地记忆 → 多 agent + 共享记忆 → 跨团队 agent 记忆市场。

### 趋势 2：Agent 安全从"讨论"走向"工具化"

Capital One 开源 **VulnHunter**（agentic 代码安全扫描）、**Kastra**（Claude Code/Cursor/Codex 策略强制执行）、**mcp-audit**（70 个 MCP server 沙箱审计）——AI agent 安全正在从白皮书走向实际工具。尤其 VulnHunter 来自 Capital One（高度监管的金融业），说明企业级 agent 安全需求已经从"要不要做"变成"怎么做"。

## 💎 本周最值得试的新技能

**如果你在用 Claude Code + skills：试试 Ratel 的按需加载思路**

即使不直接用 Ratel，也可以手动精简 skills：只保留项目根目录 `.claude/skills/` 下当前任务需要的 5-8 个核心 skill，把其他的挪到 `~/.claude/skills-archive/` 按需软链接回来。实测可节省 30-50% 上下文 token 消耗（取决于 skill 数量）。

**如果你关心 agent 安全：跑一下 mcp-audit**

`mcp-audit`（[github.com/BhaveshThapar/mcp-audit](https://github.com/BhaveshThapar/mcp-audit)）在沙箱中运行你的 MCP server 并记录所有行为——文件访问、网络请求、命令执行。70 个 MCP server 的审计数据公开可见，帮你判断哪些 server 值得信任。
