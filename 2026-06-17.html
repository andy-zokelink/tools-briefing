dongshuyan/compass-skills（司南）— 首个中文 Agent Skills 任务总控系统

司南是一个个性化 AI 任务总控 Skills 系统，自称 "Personal Alignment Skills OS for AI Agents"。它把 Agent Skills 从"让 agent 做某件事"升级为"帮人管理所有 agent 任务"——包括任务管理、知识库、记忆系统，采用 local-first 架构。创建于 6 月 15 日，2 天破 260 星。

核心信息

仓库dongshuyan/compass-skills
Stars⭐260（2 天）

标签agent-skills, ai-agents, claude-code, task-management, local-first, personal-ai
判标⚡ 值得关注进度（方向对但刚创建2天，需要更多实践验证）

为什么重要：这不是又一个"让 agent 懂某个 API"的 Skills——这是把 Agent Skills 变成"任务操作系统"。上周 ponytail 证明了 Skills 可以改变 agent 的行为范式（怎么做），本周 compass 证明了 Skills 可以成为 agent 的工作流编排层（做什么+优先级）。两条线汇聚意味着 Skills 正在从 function-attachment 变成 agent 的"人格+工作流"层。

中文生态的里程碑意义：

  • 这是中文社区首个系统化的 Agent Skills 编排方案——之前 Skills 主要由英文社区推动
  • "任务总控"定位填补了 Skills 生态的空位：之前 Skills 都是"让 agent 做 X"，但没有人做"帮人管理 agent 做哪些 X"
  • local-first 架构意味着数据主权——对于需要本地运行的中文场景特别重要
判标说明：⚡ 方向对但非常早期（2天前创建）。如果它真的能成为"Agent Skills OS"，价值巨大——但目前只有 260 星，需要观察它能否像 ponytail 一样找到 viral loop。中文开发者值得关注。
02

ruvnet/agent-harness-generator — Agent Harness 的"脚手架生成器"

这不是一个新的 agent harness——而是生成 harness 的 meta-harness。输入你的需求，自动 scaffold 出带 npx CLI、MCP server、记忆系统、学习循环和见证签名发布的完整 agent harness。支持 Claude Code、Codex、pi.dev、Hermes、OpenClaw 等 6+ 平台。创建于 6 月 13 日，4 天 156 星。

核心信息

仓库ruvnet/agent-harness-generator
Stars⭐156（4 天）
语言TypeScript
标签agent-harness, agent-scaffolding, mcp, hermes-agent, multi-agent, autonomous-agents
判标🔥 值得马上试（做 agent 平台/工具的人）/ ⚡ 关注其对标准化的推动

为什么重要：这类似于 Web 开发的历史：从手写 HTML（第一代）→ 出现框架（Rails/Django，第二代）→ 出现脚手架生成器（create-react-app / Rails scaffold，第三代）。agent-harness-generator 标志着 Agent 开发进入了"脚手架时代"——创建 agent harness 不再需要从零搭建，而是可以用模板快速生成。基础设施的抽象层级在迅速上升。

对生态的影响：

  • 降低 agent 平台创业门槛——理论上一个人用一个下午就能 scaffold 出自己的 agent 平台
  • 推动格式标准化——生成的 harness 遵循统一的 CLI/MCP/memory 模式
  • 加速垂直 agent 平台涌现——不同行业可以快速生成定制 agent harness
判标说明：🔥 如果你在维护任何 agent 工具或想搭建自己的 agent 平台，值得马上试试。它支持 Hermes Agent，理论上可以直接生成 Hermes 兼容的脚手架。⚡ 对于观察者，关注它能否推动 agent 基础设施的格式标准化——这比单个工具更重要。

📊 趋势洞察

趋势 1

Agent Skills 从"单点能力"走向"操作系统"

上周 ponytail 展示了 Skills 可以改变 agent 的"行为范式"（怎么做），本周 compass-skills 展示了 Skills 可以成为 agent 的"任务操作系统"（做什么+优先级）。两条线汇聚：Skills 正在从 function-attachment 变成 agent 的"人格+工作流"层。

传导链：Anthropic 定义 agentskills.io 标准格式 → 开发者创造功能型 Skills（懂 API、懂框架）→ ponytail 开创行为型 Skills（改变 agent 思考方式）→ compass-skills 开创 OS 型 Skills（编排 agent 任务）。从 Anthropic 定义标准到出现"个人 Skills OS"，只用了不到半年——Skills 的进化速度远超预期。

下半年预判：会看到更多"编排型"Skills 出现——不只是让 agent 做什么或怎么做，而是定义 agent 在什么场景下用哪个 Skills、按什么优先级。Skills 之间的互操作性将成为新战场。

趋势 2

Agent 基础设施进入"工业化"阶段——安全获产业背书 + 开发工具链标准化

两个信号汇聚：(1) NVIDIA SkillSpector 以硬件巨头身份进入 Agent Skills 安全赛道，产业资本正式确认了这个方向的价值；(2) agent-harness-generator 让创建 agent 平台像脚手架一样简单——从"手写"到"模板"，基础设施的抽象层级在迅速上升。

为什么这个时机很重要：上周的 hol-guard（AI 杀毒软件）+ 本周的 NVIDIA SkillSpector，表明 Agent Skills 安全已从"爱好者项目"升级为"产业赛道"。同时 agent-harness-generator 代表开发工具链的标准化——安全性标准化和开发工具链标准化在同步推进，这是 agent 生态成熟的双引擎。

下半年预判："Agent Platform-as-a-Service"（APaaS）将成为新品类——不只是单个 agent 的托管，而是 agent 平台自身的生成和运维。agent-harness-generator 是这个方向的先行者。未来可能出现 "Vercel for Agents"——一键部署完整 agent 平台。
