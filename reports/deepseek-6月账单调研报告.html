整改建议

  • 1

    立即：在 DeepSeek 控制台设置每日限额

    为"曼妮"、"阿森"等所有 Claude Code 使用的 key 设置每日消费上限（建议 ¥50–100/天），触发上限自动停用，防止下次再出现单日 ¥1,200 的情况

  • 2

    限制 Claude Code 自主运行的 turns 上限

    在 Claude Code 配置或调用命令中添加 --max-turns 100（或更小值），避免"2000 turns"这类无上限授权。每次 delegate_task 时明确告知 turns 上限

  • 3

    为 Claude Code 单独创建预算 Key

    把 Claude Code 的 ANTHROPIC_API_KEY（在 ai-api.kkidc.com 用的那个）与 Hermes 自己的 key 分开，给 CC 专属 key 设更严格的限额，出问题时不影响 Hermes 正常运行

  • 4

    在 Hermes 会话中增加费用感知提示

    当用户要求 "N turns" 时，Hermes 应先估算费用范围并告知用户。例如：100 turns × 平均 43K tokens × ¥3/M ≈ ¥13，让用户在授权前有成本预期

  • 5

    通知阿森同步设置限额

    阿森的机器也有相同配置，6月19-20日共花了 ¥1,046，应同步在 DeepSeek 控制台给"阿森" key 设置每日限额
